# 📸 Zectrix NAS Photo Pusher (桌面时光机 - 进阶版)

拒绝让 NAS 里的几十万张照片吃灰！这是一个专为 **极趣实验室 (Zectrix Lab)** AI 墨水屏便利贴（如 Note4）设计的“历史上的今天”照片推送系统。

通过打通群晖 (Synology) NAS 和 Zectrix 开放平台，你可以把放在角落吃灰的 NAS 变成一台“桌面时光机”，每天在桌面上随机重温过去的美好瞬间。

---

## ✨ 核心特性：为什么它是“进阶版”？

* 🛡️ **垃圾图片免疫（EXIF 提纯）**：独家避坑设计！彻底屏蔽群晖恶心的 `@eaDir` 缩略图文件夹。通过底层读取相机的 EXIF 元数据，精准剔除表情包、网络截图和系统缓存，只把**真正的照片**送上屏幕。
* 🎲 **公平的时光穿梭（按年份加权）**：不仅局限于“今天”，脚本会在 **前后 ±10 天** 内寻找回忆。独创的“先抽年份，再抽照片”逻辑，完美解决某一天狂拍1000张导致其他年份无法曝光的痛点，确保每一年的记忆都有平等的展示机会。
* 📐 **墨水屏专属排版**：自动读取 EXIF 纠正竖拍照片“躺平”问题，采用 LANCZOS 算法进行等比满屏居中裁剪。并在右下角自动生成一枚高对比度的小黑块铭牌打上日期，在 E-ink 屏幕上极具复古质感。

---

## 🛠️ 环境准备

在开始使用前，请确保您的 NAS 环境满足以下要求：

1. **Python 3 环境**：建议在群晖套件中心安装官方 Python 3 套件。
2. **安装依赖库**：在终端执行以下命令（注意替换为您 NAS 实际的 Python 版本，例如 `python3.14`）：
   ```bash
   python3.14 -m pip install Pillow requests
   ```
3. **获取 API 与字体**：
   * 登录 [极趣云平台](https://cloud.zectrix.com) 获取设备的 **MAC 地址** 和 **API KEY**。
   * **⚠️ 重要**：脚本右下角的日期铭牌使用了 `zfull-GB.ttf` 字体。请自行准备该中文字体文件，并与 Python 脚本放在**同一个文件夹**下，否则中文可能会显示为方块。

## 🚀 部署指南 (两步走战略)

本项目采用**“建库”与“推流”分离**的架构。

### 步骤一：运行扫描机，建立“岁月史书”索引
该脚本会遍历你的图库，提取真实的 EXIF 拍摄时间，并生成一个极速读取的 `.txt` 索引文件。

1. 打开 `build_index_demo.py`，配置以下路径：
   ```python
   # 注意务必使用绝对路径
   PHOTO_DIR = "/volume1/photo" # 你的群晖相册总目录
   INDEX_FILE = "/volume1/web/zectrix_photo_index.txt" # 索引文件存放位置
   ```
2. 在终端中运行建库脚本（首次运行时间取决于照片数量）：
   ```bash
   cd /你的脚本所在目录/ && python3 build_index_demo.py
   ```
   *运行成功后，会提示“建库完成！成功避开所有缩略图，共提纯 XXX 张高清原图”。*

### 步骤二：运行推图脚本，发送至墨水屏
该脚本负责从刚才建好的索引中，运用高级算法抽取照片、裁剪排版并推送到屏幕。

1. 打开 `zectrix_push_demo.py`，配置接口与路径：
   ```python
   INDEX_FILE = "/volume1/web/zectrix_photo_index.txt" # 必须与建库脚本填的一致
   OUTPUT_DIR = "/volume1/web" # 处理后的图片暂存目录
   
   API_URL = "https://cloud.zectrix.com/open/v1/devices/[你的设备MAC]/display/image"
   API_KEY = "你的极趣云平台API_KEY"
   ```
2. 测试推送：
   ```bash
   cd /你的脚本所在目录/ && python3 zectrix_push_demo.py
   ```
   *只要确保目录下有 `zfull-GB.ttf` 字体，几秒钟后，你的 Note4 屏幕上就会浮现出一张排版精美的老照片了！*

## ⚙️ 自动化运转 (群晖任务计划)

建库和推图测试成功后，我们将它们加入群晖的定时任务，实现完全自动化。

1. 进入群晖后台 **控制面板** -> **任务计划** -> **新增** -> **计划的任务** -> **用户定义的脚本**。
2. **建库任务（低频）**：
   * 频率建议：每周或每天半夜运行一次（用于收录新拍的照片）。
   * 运行命令：`cd /volume1/web/ && python3 build_index_demo.py`
3. **推图任务（高频）**：
   * 频率建议：每天早晨 8 点，或者每隔 2 小时运行一次。
   * 运行命令：`cd /volume1/web/ && python3 zectrix_push_demo.py`

## 💥 避坑指南：为什么不直接搜 `.jpg`？

如果你自己写过类似脚本，一定踩过这个大坑：群晖系统为了加速网页端相册，会在每个文件夹下隐蔽地生成一个叫 `@eaDir` 的文件夹，里面塞满了低清缩略图。除此之外，微信缓存、系统报错截图等也都是 `.jpg` 格式。

如果直接暴力搜索 `.jpg` 并推送到屏幕，你的“时光机”很可能会在某天清晨，给你推送一张模糊的斗图表情包。

**本项目的解法：**
`build_index_demo.py` 中内置了“免疫护盾”。不仅在路径遍历时直接拉黑 `@eaDir`，还必须通过 `PIL` 底层解析出照片的 `DateTimeOriginal`（相机/手机原始拍摄时间）和机器型号。**只有带有 EXIF 灵魂的图片，才配登上我们的墨水屏！**
